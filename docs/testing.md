# DuckBotOS — ISO Testing Guide

> How to test DuckBotOS ISO builds: VM setup, verification, boot testing, and CI.
> Status: Draft v0.1 — 2026-06-29

---

## Overview

Testing the DuckBotOS ISO happens in three stages:

| Stage | Where | What |
|-------|-------|------|
| **Verify** | Mac mini (host) | Checksum, size, structure |
| **Boot** | UTM VM (Ubuntu 24.04) | Live mode, installer |
| **Install** | UTM VM (fresh disk) | Post-install, modes |

All testing requires the Linux build VM to first produce an `.iso` file.

---

## Phase 1: Verify the ISO (Mac mini)

Before touching the VM, validate the build artifact:

```bash
# 1. Checksum (run on Mac mini after `make iso`)
cd ~/Desktop/DuckBotOS/cx-distro
sha256sum live-image-amd64.tar.gz.iso

# Compare to the .sha256 file generated by the build:
cat live-image-amd64.tar.gz.iso.sha256

# 2. Size sanity check
ls -lh live-image-amd64.tar.gz.iso
# Expected: 1.5–3.5 GB for a full desktop OS ISO
# If < 500 MB → rebuild (missing packages)
# If > 6 GB → check for accidentally bundled GPU drivers

# 3. ISO structure check (no VM needed)
# Mount the ISO and verify key files exist:
7z l live-image-amd64.tar.gz.iso | grep -E "(EFI|boot|casper|install)"
# Expected: EFI/ directory, boot.cat, casper/ filesystem.squashfs

# 4. Copy to VM share for testing
cp live-image-amd64.tar.gz.iso ~/Desktop/DuckBotOS-vm/
```

## Phase 2: UTM VM Setup (once per machine)

The build VM runs **Ubuntu 24.04 Server** (no GUI) on Mac mini via UTM.

### Create the VM

1. Download Ubuntu 24.04 LTS Server ISO: [ubuntu.com/download/server](https://ubuntu.com/download/server)
2. In UTM: **Create New** → **Virtualize** → **Linux**
3. Resources (minimum for ISO testing):
   - **Cores:** 4 (M-series chip advantage — allocation)
   - **Memory:** 8 GB RAM
   - **Disk:** 64 GB (qcow2, sparse)
4. Boot from the Ubuntu Server ISO
5. Install with defaults (openssh-server recommended)
6. After install, shut down and remove the ISO from boot order

### Share the ISO into the VM

```bash
# On Mac mini: create a shared directory
mkdir -p ~/Desktop/DuckBotOS-vm/share
cp ~/Desktop/DuckBotOS/cx-distro/live-image-amd64.tar.gz.iso \
   ~/Desktop/DuckBotOS-vm/share/

# In UTM VM config: add a VirtIO folder share pointing to ~/Desktop/DuckBotOS-vm/share
# Mount inside VM:
sudo mkdir -p /mnt/share
sudo mount -t virtiofs share /mnt/share
ls /mnt/share/  # should show the .iso
```

### Install build dependencies inside VM

```bash
sudo apt update && sudo apt install -y \
  live-build debootstrap squashfs-tools xorriso \
  isolinux syslinux-efi grub-efi-amd64-bin mtools \
  dosfstools dpkg-dev debhelper fakeroot gnupg \
  7zip  # for ISO inspection
```

## Phase 3: Boot Testing — Live Mode (VM)

**Goal:** Verify the ISO boots to the DuckBotOS splash/picker.

### Boot the ISO

1. Attach the ISO to the VM as a CD-ROM drive (UTM GUI)
2. Start the VM — it boots from the ISO automatically
3. You should see the **DuckBotOS Plymouth boot screen** → **mode picker**:
   - 🦆 **Hermes-only** — full AI agent workstation
   - 🤖 **OpenClaw-only** — CLI-focused agent OS
   - ⚡ **Hybrid** — both, selectable at login

### Live mode checks

- Plymouth theme shows "DuckBotOS" (not CX Linux)
- Mode picker is readable on 800×600 minimum
- No critical errors in boot logs (TTY4: `Alt+F4`)
- Network comes up ( DHCP on enp0s*)
- After mode selection: Weston starts, Chromium kiosk launches
- `lms` command available (LM Studio CLI)
- `browseros-cli --version` works
- `openclaw --version` or `hermes --version` works

```bash
# Inside live session (TTY2: Alt+F2):
lms version        # should print LM Studio CLI version
browseros-cli --version
journalctl -b | grep -E "(duckboto|hermes|openclaw|lmstudio)" | head -20
```

---

## Phase 4: Installation Testing (VM)

**Goal:** Install DuckBotOS to the VM's virtual disk and verify it boots standalone.

### Run the Installer

1. From live mode: double-click **Install DuckBotOS** (or `ubiquity` from terminal)
2. Installer steps:
   - **Keyboard:** defaults
   - **Network:** DHCP assumed — skip if offline
   - **Storage:** "Erase disk and install DuckBotOS" (VM virtual disk)
   - **Profile:** set a temporary password
   - **Mode selection:** pick one mode (Hermes-only recommended for first test)
3. Wait for installation to complete (~10–20 min in VM)
4. **Reboot** — remove ISO from CD-ROM drive first

### Post-install checks

```bash
# TTY2 login (temp password from install):
# Expected: new system prompt
uname -a   # should show DuckBotOS kernel, Ubuntu 24.04
cat /etc/os-release | grep PRETTY_NAME  # "DuckBotOS 24.04 LTS"

# Services
systemctl status hermes        # active (running) or failed
systemctl status openclaw      # active or failed
systemctl status lms           # active or failed
systemctl status browseros    # active or failed

# Weston + Chromium kiosk
systemctl status duckbotos-kiosk
ps aux | grep -E "(weston|chromium)" | grep -v grep

# LM Studio
lms ps          # should show no models loaded (fresh install)
curl http://127.0.0.1:1234/v1/models  # LM Studio API — should return model list

# Mode picker (GDM)
systemctl status gdm3
# Check /usr/share/xsessions/ has duckbotos-hermes.desktop, etc.
ls /usr/share/xsessions/

# AppArmor (if enforced)
aa-status | head -20
```

### Test all three modes

Repeat Phase 4 for each install mode. Each mode should:
- Boot to its respective session (Hermes/OpenClaw/GDM picker)
- Show correct Plymouth theme
- Start correct services
- Surface correct app defaults (dashboard vs CLI)

## Phase 5: Package-Level Tests (in installed VM)

After base OS verified, test individual DuckBotOS packages:

```bash
# DuckBotOS package presence
dpkg -l | grep duckbotos

# Each package installs its service
systemctl status duckbotos-base
systemctl status duckbotos-hermes
systemctl status duckbotos-openclaw
systemctl status duckbotos-lm-studio
systemctl status duckbotos-browseros

# LM Studio headless server
systemctl status lms
curl http://127.0.0.1:1234/v1/models

# BrowserOS MCP server
systemctl status browseros-mcp
# Check port 9003
ss -tlnp | grep 9003

# computer-use service
systemctl status duckbotos-computer-use

# Uninstall test (critical for release)
sudo apt purge duckbotos-meta
# Verify no leftover files in /opt/duckbotos/ or /usr/local/duckbotos/
sudo apt autoremove --purge
```

## Phase 6: Automated Test Scripts

### Run cx-distro's existing test script (adapt for DuckBotOS)

```bash
# Inside the installed VM:
git clone https://github.com/Franzferdinan51/cx-distro
cd cx-distro/tests

# Adapt for DuckBotOS (replace cx → duckbotos):
sed -i 's/cx-core/duckbotos-base/g; s/cx-full/duckbotos-meta/g' \
    installation-tests.sh

# Run smoke tests
sudo ./installation-tests.sh clean gpg repo
```

### DuckBotOS-specific smoke test

Create `~/Desktop/DuckBotOS/cx-distro/tests/duckbotos-smoke.sh`:

```bash
#!/bin/bash
set -euo pipefail
RED='\033[0;31m'; GREEN='\033[0;32m'; NC='\033[0m'
PASS=0; FAIL=0

check() {
  local cmd="$1" desc="$2"
  echo -n "Testing: $desc... "
  if eval "$cmd" &>/dev/null; then
    echo -e "${GREEN}PASS${NC}"; ((PASS++))
  else
    echo -e "${RED}FAIL${NC}"; ((FAIL++))
  fi
}

check "systemctl is-active gdm3"          "GDM3 running"
check "systemctl is-active lms"            "LM Studio daemon"
check "systemctl is-active browseros-mcp" "BrowserOS MCP"
check "command -v lms"                     "lms CLI available"
check "command -v browseros-cli"           "browseros-cli available"
check "[ -f /usr/share/xsessions/duckbotos-hermes.desktop ]" "Hermes .desktop"
check "[ -f /usr/share/xsessions/duckbotos-openclaw.desktop ]" "OpenClaw .desktop"
check "[ -f /usr/share/xsessions/duckbotos-hybrid.desktop ]" "Hybrid .desktop"
check "curl -sf http://127.0.0.1:1234/v1/models > /dev/null" "LM Studio API"

echo ""
echo "Results: $PASS passed, $FAIL failed"
[ "$FAIL" -eq 0 ]
```

```bash
# Run it:
chmod +x tests/duckbotos-smoke.sh
sudo ./tests/duckbotos-smoke.sh
```

## Phase 7: GitHub Actions CI (Post-Build)

Once an ISO is produced, automate verification in CI:

```yaml
# .github/workflows/test-iso.yml
name: Test DuckBotOS ISO
on:
  workflow_dispatch:
    inputs:
      iso_path:
        description: 'Path to ISO file'
        required: true
        type: string

jobs:
  vm-test:
    runs-on: ubuntu-24.04
    steps:
      - uses: actions/checkout@v4
      - name: Run smoke tests
        run: |
          chmod +x tests/duckbotos-smoke.sh
          sudo ./tests/duckbotos-smoke.sh
```

## Common VM Issues

| Symptom | Fix |
|---------|-----|
| VM boots to black screen | Disable Secure Boot in UTM UEFI settings |
| ISO won't attach as CD-ROM | Use UTM IDE/CD-ROM, not VirtIO |
| Slow install in VM | Increase VM RAM to 8 GB, enable host CPU cores |
| GDM not starting | Check `journalctl -u gdm3 -b`, check .desktop files |
| LM Studio API 404 | Run `lms server start` first; check port 1234 |
